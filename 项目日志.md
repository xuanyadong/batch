# 项目日志

## 2025-03-10

### 合同批量生成功能

新增可执行 JAR 包，用于根据数据表批量生成合同文件。

**实现内容：**

1. **依赖**
   - 添加 Apache POI 5.2.5（poi、poi-ooxml）用于读写 Excel
   - 添加 spring-boot-configuration-processor 支持配置绑定

2. **配置类 `ContractConfig`**
   - `data-path`: 填写数据表路径（第一个 sheet 为数据源）
   - `template-path`: 模板文件路径
   - `output-dir`: 生成文件的输出目录
   - `output-file-name-pattern`: 输出文件名模式，使用 `${列名}` 占位符

3. **核心服务 `ContractGeneratorService`**
   - 从数据表第一个 sheet 读取数据（第一行为表头）
   - 每行数据填充模板中的 `${列名}` 占位符
   - 批量生成合同文件到输出目录
   - 支持 .xls 和 .xlsx 格式

4. **启动执行**
   - `ContractGeneratorRunner` 实现 CommandLineRunner，jar 启动后自动执行批量生成
   - 执行完成后进程退出

**使用方式：**

```bash
# 使用默认配置（需在 application.yaml 中配置路径）
java -jar batch.jar

# 命令行覆盖配置
java -jar batch.jar --contract.data-path=D:/data/0126.xlsx --contract.template-path=D:/templates/塑料合同模板.xlsx --contract.output-dir=D:/output
```

**配置说明：**
- 模板中占位符格式：`${列名}`，列名需与数据表第一行（表头）一致
- 输出文件名模式示例：`合同-${买方}-${卖方}-${合同编号}.xls`

### 路径配置更新

### 文件名重复修复

- 问题：当 output-file-name-pattern 中的占位符与数据表列名不匹配时，占位符替换为空，导致所有文件互相覆盖
- 修复：文件名重复时自动追加 `_行号`；新增 `${A}`、`${B}` 列字母占位符（A=第1列，B=第2列...）
- 默认文件名模式改为 `合同-${A}-${B}-${C}.xls`

### 内容填充与自动映射

- 新增 `sample-filled-path` 和自动推断映射：对比已填充样本与 0126 第一行数据建立列→单元格映射
- 新增 `column-cell-mapping`：按“单元格=列”映射直接写入，适用于无占位符的模板，如 `B3=A,D5=B,E7=C`

### 数值类型与公式计算

- 填充时数值用 `setCellValue(double)` 写入，确保 SUM 等公式能正确计算；移除人民币大写代码转换（C15 由模板公式处理）

### 公式保留

- 仅修改含占位符 `${xxx}` 的公式单元格；不含占位符的公式单元格保持原样
- 保存前调用 `FormulaEvaluator.evaluateAll()` 计算公式；并设置 `setForceFormulaRecalculation(true)` 让 Excel 打开时强制重算（C15 中文大写等 POI 不支持的函数由 Excel 正确计算）
